──────────────────────────────────────────────────────────────────────────
1) The container image
   This chart expects the image "{{ include "agenticbear.image" . }}".
   Build & push it from the repo root, then set image.repository/image.tag:

     docker build -t <registry>/agenticbear:{{ .Chart.AppVersion }} .
     docker push    <registry>/agenticbear:{{ .Chart.AppVersion }}
     helm upgrade --install {{ .Release.Name }} deploy/helm/agenticbear \
       --set image.repository=<registry>/agenticbear

2) Reach the app
{{- if .Values.ingress.enabled }}
   http://{{ .Values.ingress.host }}/   (Ingress)
{{- else if eq .Values.service.type "LoadBalancer" }}
   Wait for the external IP:
     kubectl get svc {{ include "agenticbear.serverName" . }} -n {{ .Release.Namespace }} -w
{{- else }}
   Port-forward:
     kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "agenticbear.serverName" . }} 8080:{{ .Values.service.port }}
   then open http://localhost:8080/
{{- end }}

3) Gateway (OpenAI-compatible) — same host, `/v1` path:
{{- if .Values.ingress.enabled }}
     https://{{ .Values.ingress.host }}/v1
{{- else }}
     http://localhost:8080/v1     (after `kubectl port-forward` from step 2)
{{- end }}
   Create a gateway key in Gateway → API Keys, then point any OpenAI SDK at it.

4) First login
     User: {{ .Values.auth.adminUsername }}
{{- if eq .Values.auth.adminPassword "admin" }}
     Pass: admin        ← ⚠ default password; change it after login.
{{- else }}
     Pass: (auth.adminPassword you set)
{{- end }}
   Then rotate in Settings → Users, and for real deployments set a strong
   auth.secret via `helm upgrade` to invalidate any existing tokens.

5) Recover the AUTH_SECRET after install (needed to decrypt existing sessions):
     kubectl -n {{ .Release.Namespace }} get secret {{ include "agenticbear.secretName" . }} \
       -o jsonpath='{.data.AUTH_SECRET}' | base64 -d
